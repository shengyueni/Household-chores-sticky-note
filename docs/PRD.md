## 1. 产品定位

### 1.1 我们解决什么

家庭中存在大量**隐形劳动**：察觉「药快没了」、协调「谁去医院」、半夜下单、记着燃气年检、孩子家长群消息……这些事**没人计数、没人感谢、没人替补**，最终全压在一个人身上（多数是夹心层女性，如唐宁）。

主流家庭 todo 工具方向错了：它们让**主负责人更高效地做完更多事**。结果是她做得越多，越没人接手，**家庭分工更失衡**。

### 1.2 我们怎么不一样

家务便利贴**不优化个人完成时间**。优化目标是**家庭基尼系数**——劳动分布的平衡度。
- 主负责人做得越多 → 基尼越高 → 系统越「红」 → 自动把任务分流给其他成员
- 系统替家庭**记 / 算 / 分 / 兜**，不替她**做更多**

### 1.3 为什么用「便利贴」

- **便利贴**=轻量、随手贴、可撕可换、谁都看得见
- 对应核心交互：任何人随手「贴」一个家务（语音/视频/截图），系统自动「贴到该贴的人桌上」
- 视觉记忆点强，反「冷冰冰任务管理器」刻板印象

## 2. 设计基调（核心约束，全团队遵守）

> 主题沉重，产品要轻。

### 2.1 三条铁律

1. **不焦虑化**：不用红色警报、不用倒计时、不用「逾期」字眼。家务超时→「便利贴掉地上了，谁帮捡一下？」
2. **不审判化**：负担分高 ≠ 受害者，负担分低 ≠ 逃避者。系统呈现差距，但不贴标签。
3. **不打卡化**：复盘自愿、可关闭。打开是温柔的看见，不是 KPI。

### 2.2 语言风格

| 场景 | 不要这样写 | 要这样写 |
|------|-----------|----------|
| 任务推送 | 「您有 1 条紧急待办」 | 「妈那边降压药快见底了，谁顺手？」 |
| 升级提醒 | 「弟未及时认领，已升级」 | 「弟今天可能在忙，便利贴贴到爸桌上啦」 |
| 基尼变化 | 「失衡度下降 0.13」 | 「家里的天平今天稳了一点」 |
| 未完成 | 「逾期未完成」 | 「这张便利贴还没贴到家，等谁路过」 |
| 半夜任务拦截 | 「已屏蔽通知」 | 「夜里了，这件事让明天的人接住」 |

### 2.3 核心词汇表（统一口径）

- 任务 → **便利贴**
- 接棒 → **揭贴**（揭起便利贴=认领）
- 升级 → **传贴**
- 基尼系数 → **天平**（家里的天平）
- 心智账本 → **小账本**
- 升级到外部 → **请人帮一把**

## 3. 视觉与品牌系统

### 3.1 Q 版家庭形象

5 个固定角色 IP，全程贯穿产品 + 演示 + 周边：

| 角色 | 形象关键词 | 视觉锚点 |
|------|-----------|----------|
| 唐宁 | 抱着便利贴本子的睡眠不足上班族 | 黑眼圈 + 总在写字 |
| 弟 | 戴耳机的程序员 | 桌上有按钮和咖啡 |
| 爸 | 退休爱遛弯的实在人 | 帆布袋 + 步数高 |
| 妈 | 慢病但热爱生活的奶奶 | 拐杖 + 鲜花 |
| 「贴贴」吉祥物 | 拟人化便利贴方块小人 | 黄色 + 卷边 + 表情丰富 |

吉祥物「贴贴」=系统化身。它说话、它贴贴、它揭贴。所有提醒**由「贴贴」发出**，而不是冷冰冰系统。

### 3.2 色彩

- 主色：**便利贴黄** #FFD86B
- 辅色：**奶白** #FFF8EC、**米灰** #E9E2D3
- 强调：**绿（已揭）#7BC47F** / **粉（关心）#FFB3B3**
- 严禁：纯红警报、纯黑、霓虹色

### 3.3 字体与排版

- 中文：手写感字体（如汉仪小麦体）做标题，正文常规黑体
- 数字：圆体加粗，避免冰冷数据感
- 大量留白，模仿真实便利贴墙

### 3.4 音效

- 任务到达：纸张「啪」一声轻响
- 揭贴：「咻」轻快声
- 复盘开启：温柔短铃
- 严禁：尖锐提示音、震动连发

## 4. 目标用户与用户故事

### 4.1 角色表

| 角色 | 描述 | 在系统中作用 |
|------|------|--------------|
| 唐宁 | 夹心层主负责人，35 岁，工作+照护双线 | **被保护者**，不是 power user。系统首要任务是替她过滤、转移负担 |
| 弟 | 32 岁，工程师，独立但常被「收到」掩盖 | 接棒首选，物理按钮 + 电子纸卡推送主体 |
| 唐宁丈夫 | 协作者，与唐宁共担夜间紧急 | 夜间路由次选，避免唐宁独自起夜 |
| 爸 | 65 岁，退休，时间充裕，能跑腿 | 白天接棒主力，复诊陪同核心 |
| 妈 | 62 岁，慢病，被照护对象 | 主要事件源（药品/复诊触发），复盘卡受益人 |
| 系统（贴贴） | 替家庭记 / 算 / 分 / 兜 | 兜底者，永远在主负责人前面 |

### 4.2 核心用户故事

**故事 1（半夜药）**：
> 周二 23:47，妈在群里发「药快没了」。传统模式：唐宁凌晨 1:23 下单。本系统：贴贴拦下唐宁通知，把便利贴贴到弟桌（电子纸亮黄），弟早晨揭贴，9 点药已下单。唐宁睡了一整觉。

**故事 2（复诊接力）**：
> 妈周三复诊。传统模式：唐宁请假全程陪同。本系统：6 节点拆分——爸接「早晨陪同」（退休时间充裕），弟接「医院问诊+取药」（公司离医院近，午休跑），唐宁接「晚间报销」（半小时搞定）。一家四口接 6 棒。

**故事 3（寄生劳动曝光）**：
> 唐宁某次客户会议纪要的右上角写了「妈复诊 周三 空腹」。系统 OCR 扫到，标记「寄生劳动」，写入小账本。月底复盘：「你这个月在工作笔记里藏了 17 条家务」。

## 5. 核心功能模块

### 5.0 游戏化层（统领全局）

> 把沉重的「分工」包装成轻松的「家庭小游戏」。

**机制**：
- 5 名家庭成员以 Q 版形象进入系统
- 每月统计**基尼系数变化值**（不是绝对值，是**改善度**）
- 改善度第一名（最让家里更平衡的人）获得**家庭共同奖励决策权**
  - 例：本月旅行目的地、本周菜单、周末活动安排
- 月底「家庭天平大会」：贴贴主持，回顾每人贡献，颁发决策权徽章

**反 pattern**：
- 不奖励「做得最多」（那会强化唐宁是兜底者）
- 奖励「让家更平衡」（鼓励**接棒**而非**抢做**）
- 奖励**决策权**而非物质（提升家庭归属感，不是消费刺激）

**视觉**：
- 每人一张 Q 版形象卡，背景颜色随负担分变化
- 月度大会一镜到底动画，贴贴主持

---

### 5.1 小账本 Mental Load Ledger（原心智账本）

**功能**：把看不见的家务劳动变成可量化数据。

**4 类劳动打分**：

| 类型 | 例 | 分值 |
|------|----|----|
| 察觉 | 「药快没了」「燃气该年检了」 | +1 |
| 协调 | 「我跟弟商量下」「排个时间」 | +2 |
| 执行 | 下单、跑医院、缴费 | +3 |
| 兜底 | 没人接最终兜上 | +5（曝光警示） |

**家庭天平（基尼系数）**：
- 实时计算 0-1 数值，0=完全平衡，1=极端失衡
- 大屏可视化：4 人柱状图 + Q 版形象，柱子高=负担重
- 不显示数字给老人，避免他们焦虑

**关键设计**：
- 兜底分**最高**：系统宁可不让你兜，也不让你白兜
- 「察觉」也算分：说出口=贡献。鼓励家人主动发现而非沉默观察

---

### 5.2 接棒协议 Baton Protocol

**功能**：把群里黑洞式「收到」改成可追踪的认领回执。

**结构化揭贴卡**：
```
[贴贴推送] 妈降压药快见底了
推荐：弟  ETA：明早 9 点前下单
[揭贴] [转给别人] [贴贴帮我催]
```

**未揭贴自动「传贴」（升级链）**：
- 顺序：弟 → 爸 → 唐宁丈夫 → 社区/付费跑腿 → **唐宁（永远在最末，且仅白天）**
- 上一级电子纸灭灯，下一级亮黄
- 未揭贴 -1 分入小账本（不公开羞辱）
- 群播报使用温柔语：「弟今天可能在忙，便利贴贴到爸桌上啦」

**反兜底设计**：
- 唐宁默认不在升级链中，需主动揭贴才计入
- 唐宁连续 3 次主动揭贴 → 触发「家庭天平警报」，提醒全家：天平又开始倾斜

---

### 5.3 睡眠护栏 Sleep Guard

**功能**：保护照护者夜间睡眠，让她真的能睡。

**机制**：
- 唐宁设睡眠窗（默认 23:00–7:00，可调）
- 窗内产生的便利贴**不推送给她**
- 路由优先级：
  1. **弟弟**（首要推送人，年轻，作息更弹性）
  2. **唐宁丈夫**（夜间紧急共担，避免唐宁独自起夜）
  3. **异步队列**（非紧急→明早 7 点起统一推送）
- 检测到唐宁夜间手动操作（半夜下单/查群），贴贴温柔提醒：「夜里了，这件事让明天的人接住吧」

**演示金句**：
> 「她今晚能睡了」——这是产品的最高目标。

---

### 5.4 复诊流水线 Hospital Pipeline

**功能**：把「复诊」一个大任务拆 6 节点，多人异步接棒。

**6 节点**：

| # | 节点 | 时间 | 推荐 owner 逻辑 |
|---|------|------|-----------------|
| 1 | 前夜准备（空腹通知 + 材料打包） | 复诊前晚 | 同住人 |
| 2 | 医保卡 + 病历 + 既往报告整理 | 复诊前晚 | memAura 查上次整理过的人 |
| 3 | 早晨陪同出发 | 当日 7:00 | 早晨空闲 + 路径匹配（XGRIDS） |
| 4 | 医院全程（挂号/问诊/检查） | 当日 8-12 | 时间块 ≥4h 的人 |
| 5 | 取药 + 用法记录 | 当日 12-13 | 距医院近 + 午休段可达 |
| 6 | 报销提交 + 入账 | 当日或次日 | 单位有医保对接的人 |

**多人协同 4 机制**：
1. **强制曝光**：6 节点同时推送到全家终端，唐宁侧默认按钮灰，长按 3 秒强制激活
2. **智能推荐**：每节点带「推荐：XX」标签，依据日历 + 距离 + 历史
3. **支援角色**：每节点支持多个支援者（owner ×1.0 分，支援者 ×0.5 分），鼓励多人参与
4. **节点间交接**：完成时自动 ping 下一棒 + memAura 注入上下文（医生提示/老人禁忌/账户信息）

**memAura 注入**：
- 上次复诊摘要 + 待复查项
- 老人过敏 / 慢病禁忌
- 报销账户绑定信息
- 周期性复诊间隔规律

### 5.5 零打字录入 Zero-Type Capture

**反 pattern**：让累的人打字 = 系统自己制造心智负担。

#### 录入 1：口喷（语音）
- 手机长按浮窗 / 硬件麦克风唤醒词「家事」/「贴贴」
- 边做饭/开车/哄睡随口说
- ASR + HaiSnap VoiceParser → 结构化便利贴
- 说话人 +1 察觉分（说出口=贡献被看见）

#### 录入 2：视频提取
- 家庭群视频通话 / 监护摄像头 / 随手拍
- 抽帧 + ASR + Vision LLM
- 识别药盒余量、账单、通知单、血压计读数
- 例：拍药盒空格 3 秒 → 续购便利贴自动建

#### 录入 3：微信记录提取
- 入口：截图上传 / 转发到企业微信机器人 / 每日聊天摘要转发
- OCR + ChatExtractor 还原对话流时间线
- 抽察觉/协调/执行/兜底事件
- 让群聊「收到」黑洞第一次变成可量化数据
- 不做个人微信 hook（违 ToS）

#### 录入 4：OCR 扫描往期备忘录
- 手机拍 / 旧本扫描会议纪要、便利贴、记事本
- NoteExtractor 识别「寄生在工作场景中的家务」
- 笔记主体=工作 + 边角=家务 → 标「寄生劳动」
- 追溯曝光过去几个月被吞掉的隐形劳动

#### 录入 5：每日复盘（自愿）
- 晚 10 点可选打开（**完全可关闭，无打卡焦虑**）
- 当日全事件自动汇总
- DailyDigest 生成自然摘要 + 勾选卡
- 默认全勾，用户只做减法（取消/否认）
- 复盘反哺抽取规则
- 让兜底者第一次看见自己一天做了多少事

**复盘卡示例**（语气一定要轻）：
```
贴贴帮你回忆今天 ✿

今天你贴出去 12 张便利贴
其中 9 张被家人揭走啦
还有 3 张你自己揭了——
  - 凌晨那张「下单降压药」让弟试着接住下次好不好？

家里的天平今天稳了一点点。
明天周三妈复诊，6 张便利贴已经分好了。

[都对，关掉] [我想改一下]
```

## 6. 硬件方案（极简 1 件）

### 6.1 单显示屏（核心 + 唯一硬件）

**型号**：Seeed 圆形显示屏 / Wio Terminal（现场领，二选一）
**主控**：Seeed XIAO ESP32S3（带麦克风 + WiFi）
**外设**：
- INMP441 麦克风（语音输入）
- 小喇叭（贴贴语音输出）

**功能**：
1. **语音交互**：家庭成员对它说话，自动建便利贴；它对家庭成员说话，温柔提醒揭贴
2. **常驻显示**：当前未揭贴便利贴 + 家庭天平 Q 版图（5 形象柱状）
3. **氛围光**：天平稳=黄柔光，倾斜=橙脉冲（不用红色）

**摆位**：客厅最显眼处。所有人路过都能看见 + 被语音点名。

### 6.2 为什么只用 1 件

- **设计基调**：家不是赛博实验室。1 件硬件足够承载叙事，多了会显得侵入性强、压迫感重
- **演示稳定**：单点故障可控，36h 内联调充分
- **降本**：BOM 控制在 ¥150 内，赛后家庭可负担
- **故事性**：「家里多了一个会说话的便利贴」比「家里多了一堆传感器」更亲切

### 6.3 硬件交互脚本（贴贴语音）

| 触发 | 贴贴说什么 |
|------|----------|
| 妈在群里说「药快没了」 | 「弟，妈那边药快见底啦，明早顺手下单好不好？」 |
| 弟没在 2h 内揭贴 | 「弟可能在忙，爸方便接一下吗？」 |
| 唐宁夜里查手机 | 「夜里了，这件事让明天的人接住吧。」 |
| 月底天平大会 | 「这个月让家里最平衡的人是爸，下次旅行去哪由他决定～」 |

## 8. 数据模型（概念层，无代码）

- **家庭成员**：基本信息 + 睡眠窗 + 角色权重 + Q 版形象 ID
- **便利贴**（任务）：标题、地点、类型、状态、当前 owner、ETA、来源（语音/视频/微信/OCR）
- **心智事件**：成员 × 便利贴 × 类型（察觉/协调/执行/兜底）× 分值 × 时间戳
- **揭贴回执**：便利贴 × 揭贴人 × ETA × 缺口
- **传贴日志**：便利贴 × 升级序列 × 各级响应时间
- **memAura 引用**：便利贴 × 历史上下文 token
- **房间/空间**（XGRIDS）：房间 × 路径 × 主要使用人
- **天平快照**：日 × 各成员负担分 × 基尼值 × 改善度

## 9. 用户旅程（端到端）

```
[触发]

        ↓

        ↓
[推荐揭贴人]
BatonRouter 计算：日历空闲 + 距离（XGRIDS）+ 历史经验 + 睡眠护栏
推荐 = 弟（首选）
        ↓
[推送]
单显示屏语音播报 + 全家手机轻提醒
唐宁端 = 静默（睡眠窗）/ 灰按钮（白天但非默认）
        ↓
[揭贴 or 传贴]
弟揭贴 → 小账本 +3 执行分
弟未揭贴 → 2h 后传给爸（弟 -1 分），贴贴温柔解释
        ↓
[完成]
便利贴变绿，小账本更新，天平重新计算
        ↓
[复盘]
晚 10 点（自愿）：贴贴整理今日所有便利贴流转 → 用户确认/删减
        ↓
[月度]
天平大会：改善度第一者获家庭决策权
```
