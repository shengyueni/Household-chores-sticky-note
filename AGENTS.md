# 家务便利贴 — Codex 工作守则

> Codex 进入仓库前必读。所有改动必须遵守本文件。

---

## 一、项目背景（30 秒读完）

- **黑客松**：第五夜｜待办家事（故事赛道）
- **时间窗**：2026-05-23 20:00 → 2026-05-25 12:00
- **目标奖项**：故事赛道一等奖 + Seeed 标牌 + HaiSnap 最佳 Agent + 丘脑 memAura Token + XGRIDS
- **赛题反 pattern**：不优化主负责人（唐宁）效率，优化**家庭基尼系数**——劳动分布的均衡度
- **核心受众**：夹心层照护者（如唐宁，35 岁，工作+照护双线）

完整产品文档：`docs/PRD.md`

## 二、改动前必读规则

### 2.1 视觉铁律（违反必拒）

- **主色**：便利贴黄 `#FFD86B`
- **辅色**：奶白 `#FFF8EC`、米灰 `#E9E2D3`
- **强调**：绿（已揭）`#7BC47F` / 粉（关心）`#FFB3B3`
- **严禁**：纯红警报、纯黑、霓虹色、强对比刺眼配色
- **风格**：手写感、便利贴质感、留白多、Q 版可爱

### 2.2 文案铁律

主题沉重，产品要轻。

| 不要这样写 | 要这样写 |
|------|-----|
| 您有 1 条紧急待办 | 妈那边降压药快见底了，谁顺手？ |
| 弟未及时认领，已升级 | 弟今天可能在忙，便利贴贴到爸桌上啦 |
| 失衡度下降 0.13 | 家里的天平今天稳了一点 |
| 逾期未完成 | 这张便利贴还没贴到家，等谁路过 |
| 已屏蔽通知 | 夜里了，这件事让明天的人接住 |

所有提醒以**吉祥物「贴贴」**口吻发出，不要冷冰冰系统语。

### 2.3 三条铁律

1. **不焦虑化**：不用红色警报、不用倒计时、不用「逾期」字眼
2. **不审判化**：负担分高 ≠ 受害者，负担分低 ≠ 逃避者
3. **不打卡化**：复盘自愿、可关闭，不是 KPI

### 2.4 核心词汇表（统一口径，禁止改写）

| 概念 | 必用词 |
|------|--------|
| 任务 | 便利贴 |
| 接棒/认领 | 揭贴 |
| 升级/换人 | 传贴 |
| 基尼系数 | 天平 / 家里的天平 |
| 心智账本 | 小账本 |
| 升级到外部 | 请人帮一把 |
| 吉祥物 | 贴贴 |

### 2.5 角色铁律

- **唐宁**=主负责人，**永远在升级链最末**，且仅白天
- **弟**=接棒首选
- **丈夫**=夜间紧急共担
- **爸**=白天接棒主力（退休时间充裕）
- **妈**=主要事件源（被照护对象）
- **不要**让唐宁默认接棒。系统首要任务是**减她负担**。

## 三、技术约束

### 3.1 当前 Demo 形态

- **入口**：`public/index.html`，单文件可双击跑
- **依赖**：纯 HTML + CSS + JS，**不引外部库**（除 CDN 字体可选）
- **后端**：暂无，所有交互前端 mock
- **目标**：黑客松路演时浏览器直接打开演示

### 3.2 改动范围授权

| 范围 | 授权 |
|------|------|
| UI / 视觉 / 动画 / 文案 | ✅ 自由动 |
| 交互 / 状态机 | ✅ 自由动 |
| 数据模型字段增删 | ⚠️ 改前在 commit message 说明 |
| 删除现有功能 | ❌ 禁止，除非明确指令 |
| 引入新依赖 | ⚠️ 只能 CDN，不要 npm install |
| 改 PRD.md | ❌ 禁止，PRD 是合约 |
| 改 AGENTS.md | ❌ 禁止 |
